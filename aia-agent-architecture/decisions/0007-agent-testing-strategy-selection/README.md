# 设计本功能时需要注意的点

本目录用于梳理和对比 AIAgent 系统在「测试与评估」层面的方案选择，重点关注：

- 如何系统性地验证 Agent / 工具 / 编排逻辑的正确性与稳定性；
- 如何持续评估模型与 Prompt 调整带来的行为变化；
- 如何在离线评估、在线监控、人审抽检之间做权衡；
- 如何与 0004 可观测性、0005 驭控工程协同设计一套可回放、可度量的测试体系。


在编写本目录下具体方案文档时（后续可按需拆分为多篇，如 `offline-eval-vs-online-metrics.md`、`golden-trace-vs-random-sampling.md` 等），建议围绕以下几个方面展开：

1. 测试粒度与对象
- 工具级：
  - 对 FileRead/FileEdit/HTTP 调用等工具做传统单元/集成测试，确保输入输出与错误处理符合预期。
- Agent 级：
  - 针对 Explore/Plan/Verification 等核心 Agent 设计「输入 → 期望行为」的测试用例（golden cases），验证 Prompt 与工具组合是否稳定。
- 编排级：
  - 针对完整流程（如「检索 → 规划 → 实现 → 验证」）做端到端测试，校验不同分支和异常路径。

2. 评估数据与基准集（Eval Set）
- 静态基准集：
  - 人工构造或从历史对话/任务中抽取一批具有代表性的测试样本，作为长期评估基准；
  - 需要覆盖常见路径、边界情况和高风险操作。
- 动态采样：
  - 从线上真实流量中抽样（匿名化/脱敏后）用于离线回放评估；
  - 结合 0004 的日志/Trace，选取高错误率或低满意度的样本重点关注。

3. 评估方式与指标
- 自动评分：
  - 通过规则（正则/结构化字段）、辅助模型（Judge LLM）、程序判定等方式对输出进行自动打分；
  - 指标包括正确率、通过率、一步到位率、工具使用合规率等。
- 人审与主观评价：
  - 在高价值或高风险场景，对部分样本进行人工评审，作为自动评分体系的校准依据。
- 在线指标：
  - 结合 0004，关注错误率、重试率、超时率、人工接管率、用户满意度、业务 KPI（转化率等）。

4. 回归测试与变更管理
- Prompt / 模型 / 工具 / 编排逻辑变更时：
  - 如何快速在基准集上跑一轮回归，捕获行为差异；
  - 如何利用「golden trace + 差异报告」帮助人工快速审阅变化。
- 与发布流程集成：
  - 在 CI/CD 中加入 Agent 级/编排级回归，避免无意中引入行为退化；
  - 对于高风险场景，要求通过一系列「硬门槛」测试后才能发布。

5. 安全与驭控相关测试
- 高危操作防护：
  - 设计专门测试用例，验证 Permission/Hook/防注入等驭控机制是否生效；
  - 模拟 Prompt 注入、工具参数构造攻击、长上下文污染等场景。
- 可回放与审计：
  - 确保测试与评估过程本身也被记录（Trace/日志），便于问题追踪与审计。

在后续具体文档中，建议针对以下典型方案进行深入对比：

- 仅依赖传统单元/集成测试 vs. 引入 LLM 特有的 Eval 体系；
- 纯离线基准集评估 vs. 结合线上指标与抽样人审；
- 使用开源/托管评估平台（如 LLM Evals 平台）vs. 自研轻量评估服务；
- 将测试嵌入 CI/CD vs. 通过独立评估环境周期性运行。
