# 网络搜索与开放域知识

## 1. 定义

网络搜索是指通过搜索引擎或开放 API（如通用 Web 搜索、行业垂直搜索）获取公开互联网上的最新信息，用于：

- 弥补内部知识库缺失的信息。
- 获取强时效性内容（新闻、政策更新、产品变更等）。

## 2. 适用场景

1. 时效性要求高
   - 例如需要了解最近一周的政策变化、产品公告等。

2. 开放域问题
   - 企业内部没有完整知识库的问题，可以尝试从网络获取参考答案。

3. 竞品与行业分析
   - 需要了解其他厂商产品、行业报告等公开信息。

## 3. 风险与挑战

1. 内容可信度
   - 公开网页质量参差不齐，包含广告、错误信息、过期内容等。
   - 需要对来源站点做基本信任度评估和白名单控制。

2. 合规与隐私
   - 某些行业和地区对数据采集和使用有严格限制，需要遵守当地法规和平台条款。

3. 不可控的变化
   - 搜索结果随时间变化，难以完全复现同一查询的结果。

## 4. 在 AIAgent 中的使用建议

1. 网络搜索作为补充而非主通路
   - 主通路仍应是内部知识库（文档、代码、配置、日志等），网络搜索只在知识缺口明显或需要时效性时启用。

2. 对搜索结果做最小清洗与重排
   - 去除明显无关或低质量页面。
   - 尽量提取页面中的主体内容，而非直接把整页 HTML 塞给模型。

3. 清晰标注来源
   - 在生成结果中显式展示使用了网络搜索，并列出引用的链接或站点。

4. 对高风险场景施加额外约束
   - 涉及合规、法律、医疗等敏感领域时：
     - 网络搜索结果只能作为参考，不应直接驱动自动化决策或执行。
     - 必须经过人工审核或专业系统验证。
