# 描述

依赖第三方 SaaS 平台（如 OpenRouter、Vercel AI Gateway 等）提供的统一模型接口和路由能力，把多模型管理、鉴权和扩展交给外部平台处理。近年来，这类平台通常还会提供缓存、重试、AB 实验、流量镜像、成本统计等附加能力，而不仅仅是「统一调用入口」。

# 优点

- 快速获得多模型能力：
  - 一次集成即可访问多家模型供应商和多种模型。
- 平台负责扩展与运维：
  - 多数 SaaS 平台负责基础设施的伸缩、监控和部分治理能力（限流、路由、统计）。
- 附加能力丰富：
  - 常见的第三方网关会内置请求缓存、自动重试、AB 实验、灰度发布、统一计费与成本报表等功能，减少自建时需要开发的通用能力。

# 缺点

- 供应商锁定与合规风险：
  - 被第三方平台的能力边界限制，合规和数据出境等问题需要额外评估。
- 自定义程度有限：
  - 对一些企业内部特定的审计、路由规则不一定有很好的扩展接口。
- 数据路径更复杂：
  - 引入额外一跳网关后，数据从业务系统 → 第三方网关 → 各家模型的路径更长，对隐私、审计与延迟敏感的场景需要谨慎评估。

# 使用第三方网关的开源项目示例

- pi-mono：
  - 在 `@mariozechner/pi-ai` 中内置对 OpenRouter、Vercel AI Gateway 等 Provider 的支持，展示了如何在应用层统一使用多个第三方网关。
- 各类前端/CLI 小工具：
  - 很多个人项目选择接入 OpenRouter 等平台，以较低成本获得多模型支持。
