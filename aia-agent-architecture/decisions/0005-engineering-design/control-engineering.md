# 目标

- 在保证模型发挥能力的同时，通过策略、工具与编排对其行为进行「可控的约束与引导」，确保安全、可靠、可解释。
- 将「控制」从单纯依赖 Prompt 的软约束，升级为结合权限、工具白名单、策略引擎和运行时监控的立体机制。

# 核心关注点

- 工具与权限边界：
  - 为模型提供的工具集必须最小化、白名单化，对高风险操作（写文件、大规模重构、生产变更）增加显式确认环节。
  - 不同 Agent 具备不同权限边界（只读 / 读写 / 运维级），由配置与编排层共同控制。
- 编排与策略：
  - 通过编排框架（如 LangGraph 或自研状态机）控制「调用顺序」「重试策略」「回滚路径」，而不是完全交给模型自由决定。
  - 在关键步骤增加审核节点（人工或二次模型）对结果进行检查。
- 运行时约束：
  - 对 Token、时间、调用次数、成本等资源做配额与告警，防止异常消耗。
  - 对异常行为（频繁失败的工具调用、越权访问）及时记录与拦截。
- 审计与可解释性：
  - 为每个重要决策保留「输入上下文 + 模型思考（如可用）+ 工具调用链 + 最终输出」的记录，便于事后复盘。

# 常见实践与开源项目参考

- pi-mono：
  - `@mariozechner/pi-agent-core` 通过事件流形式暴露 Agent 工作过程，便于在 UI 与日志中实现细粒度控制与审计。
- OpenCode：
  - AGENTS.md 中明确限制可执行命令、禁止泄露敏感信息、要求工具幂等与可重试，体现了对「驭控」的工程化约束。
- aiguides：
  - 对 OAuth、JWT、CORS 等安全配置有详细说明，并通过中间件控制鉴权与权限，属于「传统 Web 安全」对 LLM 系统的驭控延伸。
- 各类 Guardrails / 安全框架（如 OpenAI policy、LLM Guardrails 项目等）：
  - 提供内容过滤、敏感信息检测、输入输出检查等能力，可与 AIAgent 的驭控工程结合使用。
