# 描述

使用第三方一体化可观测性平台（Datadog、New Relic 等）采集指标、日志和 Trace，由平台提供统一的 UI、告警与分析能力。近年来，这类平台普遍支持 OpenTelemetry 接入，并开始提供针对 LLM/Agent 的专门可视化和分析能力（如对某些服务调用进行 AI Span 标注、统计 Token 消耗等）。

# 优点

- 开箱即用能力强：
  - 自带仪表盘、告警模板和强大的搜索分析能力，上线速度快。
- 深度云原生集成：
  - 对主流云厂商与 Kubernetes 生态有完备支持，采集配置简单。
- 对 LLM/Agent 场景支持逐步增强：
  - 部分平台提供与 LLM 相关的预置 Dashboard、Token 成本统计或 AI Span 类型，有助于更快搭建针对大模型应用的观测视图。

# 缺点

- 成本较高：
  - 按数据量/主机数计费，随着规模增长成本上升明显，LLM/Agent 系统往往产生大量日志和 Trace，更需要做好采样与保留策略，否则费用容易失控。
- 供应商锁定：
  - 迁移到其他方案成本较大，需要重新建设 Dashboard 和采集配置。
- 与现有策略冲突风险：
  - 在已有自建监控体系的前提下，引入新平台可能面临流程与策略冲突，需要额外治理。

# 使用 SaaS 可观测性平台的开源项目示例

- 各类云原生示例应用：
  - 很多云厂商与 SaaS 平台的官方示例会演示如何把示例应用接入 Datadog/New Relic 等服务，用于教学和展示。
