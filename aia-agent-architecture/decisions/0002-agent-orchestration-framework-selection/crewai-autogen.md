# 描述

CrewAI、AutoGen 等高层 Agent 框架为多 Agent 协作提供了「角色」「任务」「协作模式」等高层抽象，强调开箱即用的场景模板，适合快速搭建 Demo 或验证概念。

# 优点

- 上手门槛低：
  - 提供大量现成示例和模板，适合在短时间内搭出可用 Demo。
- 对典型场景支持好：
  - 对「多 Agent 分工协作」「代码评审助手」「文档助手」等常见场景有预制模式。

# 缺点

- 抽象层次较高：
  - 对于需要细粒度控制（自定义路由、复杂状态机）的场景，往往需要绕过或扩展框架，开发成本反而加大。
- 从 Demo 到生产的迁移成本：
  - 很多项目在原型验证阶段使用这类框架，上生产时则需要大量重写或迁移到更底层的抽象（如自研编排或 LangGraph）。
- 与现有基础设施整合难度：
  - 框架在日志、监控、权限等方面的扩展点有限，与企业级基础设施对接时需要额外封装。

# 使用 CrewAI / AutoGen 的开源项目示例

- 官方示例仓库：
  - CrewAI、AutoGen 自身提供了大量基于框架搭建的多 Agent Demo（代码助手、数据分析、写作助手等）。
- 社区 Demo 项目：
  - GitHub 上有不少基于这类框架构建的玩具项目或原型，用于展示多 Agent 协作，但很少作为长期维护的核心编排层。
