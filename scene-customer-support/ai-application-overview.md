## 1. 行业与产品形态概览

- 行业形态：
  - 典型代表：B2C/B2B SaaS 客服系统、全渠道联络中心（呼叫中心 + IM + 邮件）、CRM 套件中的服务模块等。
  - 客服能力逐步从「坐席工单工具」演进为「全客户旅程服务中枢」，覆盖售前咨询、售中进度查询、售后投诉与续费挽留。
- AI 原生趋势：
  - 从早期 FAQ Bot、意图识别 + 流程引导，逐步走向：
    - 大模型驱动的自然语言问答（多轮对话 + RAG）；
    - Agent 化的「自动完成部分任务」能力（改签、退款、配置查询/修改建议等）；
    - 与 CRM/工单系统深度联动的「自动摘要 + 自动推荐操作」能力。

## 2. 典型业务场景清单

- 售前咨询：产品能力、价格方案、活动政策等问答；
- 订单/服务进度查询：发货、安装、工单处理进度等；
- 故障/投诉处理：问题排查、补偿政策说明、升级路径；
- 帐号与权限问题：登录失败、权限开通、套餐升级/续费；
- B2B 的续费与扩容挽留：基于使用数据和历史工单给出续费建议。

## 3. 核心业务流程概览

- 典型流程（简化）：
  1. 用户通过 Web/App/IM/电话接入；
  2. 入口层采集上下文（用户身份、历史对话、订单/合同信息）；
  3. 路由引擎决定：自助（知识库/AI）还是直连人工；
  4. 若走自助 AI：
     - 调用知识库、订单系统、配置中心等查询；
     - 生成回答或操作建议；
  5. 若升级人工：AI 生成问题摘要、关键信息和推荐操作，辅助坐席处理；
  6. 交互结束后更新工单/会话记录，进入质量评估与复盘闭环。

## 4. 主流技术架构特征

- 核心组件：
  - 渠道接入层（Web/小程序/App/电话/IM）；
  - 路由与队列服务（排队、优先级、多技能组）；
  - 知识库与搜索引擎；
  - 工单系统与 CRM；
  - 数据平台（会话日志、客户画像、运营报表）。
- AI 相关组件：
  - LLM 接入层（直连云厂商或经网关）；
  - 知识库向量检索（RAG）；
  - 对话管理 / 会话状态服务；
  - Agent Orchestrator（决定是否调用查单、改单、升级人工等工具）。

## 5. 本场景的 AI 应用需求

- 问答与知识检索：
  - 在多语言、多业务线、多版本文档下，快速准确回答问题；
  - 需要引用来源（文档/工单/配置）以增强可解释性。
- 工单/会话辅助：
  - 对长对话自动摘要，提取关键事实和承诺条款；
  - 自动推荐标签、分类和下一步处理动作。
- 事务性操作辅助：
  - 查询订单/工单状态，并以自然语言解释；
  - 在权限和风控约束下，生成变更操作草案（改签、调整账期、补偿方案等），由人工确认后执行。
- 质量与合规：
  - 自动质检（监控敏感词、合规用语、响应时效等）；
  - 对 AI 与人工回答的质量进行统一评估与对比。

## 6. AI 带来的业务价值

- 效率提升：
  - 大量「重复 FAQ + 简单查单」类请求由 AI 解决或大幅缩短处理时间，降低人力成本；
  - 坐席平均处理时长（AHT）和排队时间下降。
- 体验改进：
  - 24/7 多渠道统一体验；
  - 回答更一致、信息更完整（自动补充历史记录和上下文）。
- 质量与管理：
  - 更高覆盖率的质检与合规监控；
  - 通过对会话和工单的统一分析，反向推动产品优化和运营策略调整。
