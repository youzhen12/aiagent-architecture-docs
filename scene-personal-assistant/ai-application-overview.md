## 1. 行业与产品形态概览

- 产品形态：
  - 面向个人或小团队的 AI 助手，典型载体为桌面/移动 App、浏览器插件、命令行工具等；
  - 职能从单一聊天问答扩展为「任务管理 + 信息检索 + 自动化脚本」的一体化工作流助手。
- 发展方向：
  - 从「智能聊天」向「智能执行」演进：更关注时间管理、项目推进、知识沉淀和自动化操作；
  - 越来越多产品在本地/近端运行，强调隐私与个性化。

## 2. 典型业务场景清单

- 信息整理：
  - 汇总邮件、会议纪要、网页内容，生成摘要和待办清单；
- 任务管理：
  - 将自然语言描述转化为任务/日程/提醒；
  - 关联项目、联系人和文件。
- 轻量自动化：
  - 自动生成模板文档、报告草稿、邮件/消息回复草稿；
  - 调用本地脚本或 API 完成重复性工作（如批量重命名、转换格式等）。

## 3. 核心业务流程概览

- 典型流程：
  1. 用户以自然语言描述目标（今天要做什么、帮我整理这堆资料等）；
  2. 助理解析意图，拉取相关上下文（日历、任务列表、文档等）；
  3. 生成任务/计划/草稿，必要时调用工具执行简单操作；
  4. 记录结果并更新长期记忆（长期任务、偏好）。

## 4. 主流技术架构特征

- 通常为：本地客户端 + 远程模型服务 / 本地模型 + 本地存储；
- 关注点：
  - 数据隐私与权限（本地文件、日历、Email 等敏感数据）；
  - 多端同步与离线可用性；
  - 简洁的工具层封装（文件、日历、任务、浏览器等）。

## 5. 本场景的 AI 应用需求

- 语义理解与规划：
  - 把模糊需求转化为可执行的计划或任务清单；
- 上下文整合：
  - 将多源信息（文件、邮件、聊天记录等）整合成统一视图；
- 工具调用与自动化：
  - 在明确边界内调用本地/云端工具执行简单操作；
- 个性化记忆：
  - 记住用户偏好、习惯和长期目标，影响后续决策与建议。

## 6. AI 带来的业务价值

- 提高个人生产力：
  - 减少在多应用之间切换和重复性操作；
- 降低心理负担：
  - 把「记住所有事情」的压力转移给系统，用户只需关注决策和执行；
- 帮助构建个人知识体系：
  - 长期积累的笔记、任务和成果通过语义索引和 Agent 协作发挥更大价值。
