## 1. 背景与挑战

- 客户类型：ToC/ToB 混合型在线客服体系，覆盖 Web / App / 小程序 / IM / 电话等多渠道，日均会话量 5–50 万级。
- 现状与痛点：
  - 高峰期排队严重：节假日、大促等高峰期会话暴涨，人工座席资源扩容成本高、排班复杂；
  - FAQ 与知识库维护困难：业务规则、活动策略、产品功能变更频繁，传统 FAQ/知识库更新滞后，导致「问不到 / 回不准」；
  - 服务质量不稳定：新客服上手周期长，话术依赖个人经验，质检覆盖率 < 3%，难以及时发现风险对话；
  - 系统割裂：客服系统、工单系统、CRM、订单系统等存在多套入口，客服需要在多个系统间频繁切换。

> 目标：在不大幅增加人工成本的前提下，提升自助转化率和问题一次解决率，同时显著提高质检覆盖率与服务一致性。

## 2. 解决方案概述

- 引入基于大模型的智能客服体系，构建「自助问答 + 工单助手 + 质检助手」一体化方案：
  - FAQ / 业务规则自助问答：基于企业知识库（政策、产品说明、活动规则等）和历史对话，支持多轮追问澄清；
  - 账号 / 订单 / 设备状态查询：通过工具调用现有订单、履约、设备系统，实现安全可控的查询与解释；
  - 升级单与总结：对复杂问题，自动生成结构化升级单、会话小结与用户画像，分发至二线 / 专家团队；
  - 质检与辅导：对全量或抽样会话进行自动质检，识别敏感话术、合规风险，并为新客服生成辅导建议。
- 技术基座（阿里云 AI MaaS）：
  - 模型层：基于 Qwen 系列对话模型（如通用对话模型 + 多语言/多模态能力），结合文本 Embedding 模型用于知识检索；
  - 平台层：使用阿里云百炼 Model Studio 的模型调用、知识库（RAG）、应用构建与插件能力，构建客服专用 Agent 应用；
  - 编排层：基于场景路由、多 Agent 协作（Router / FAQ / Task / Escalation / QC）实现端到端对话编排与决策。

## 3. 技术架构亮点

- 架构参见：`scene-customer-support/architecture-overview.md`
- 关键设计亮点：
  - 统一接入网关：各渠道会话先进入客服接入网关，由网关完成鉴权、限流、AB 实验和多模型策略分流，AI 与人工在此处无缝衔接；
  - 工具化封装：将客服系统、CRM、订单系统、履约系统等以「工具 / 插件」形式封装，对 Agent 暴露标准接口，方便在百炼应用中通过插件或 Assistant API 调用；
  - 知识库 RAG：将 FAQ、政策文档、流程手册等划分为不同知识集合，在百炼知识库中配置向量检索与分片策略，不同业务线使用独立库并结合权限控制；
  - 多 Agent 协同：
    - RouterAgent：根据意图和风险等级，路由到 FAQAgent / TaskAgent / EscalationAgent 等；
    - FAQAgent：专注知识问答与澄清；
    - TaskAgent：调用后端工具完成查询、改签、取消等操作，生成解释性回复；
    - EscalationAgent：构建升级单、总结、标签，推送至人工队列；
    - QCAgent：对对话进行合规、话术和满意度评估。
  - 安全与可观测性：通过「只读查询 + 草稿操作 + 人工确认」策略控制高风险操作，并接入日志系统做全链路追踪和指标分析。

## 4. 基于阿里云 Model Studio 的实现要点

- 模型与推理：
  - 在 Model Studio 的 Model Center 中选择合适的 Qwen 对话模型（按语言、性能、成本分层），通过 OpenAI 兼容或 DashScope 接口统一调用；
  - 使用 Qwen Embedding 模型构建知识库索引，配合向量数据库或百炼内置索引服务实现高召回率的 RAG；
- 知识库与应用构建：
  - 使用百炼「知识库（RAG）」能力，按业务线/国家/产品线划分多个知识库，支持增量更新和版本管理；
  - 在「应用开发」中定义客服场景应用，编排系统 Prompt、知识库和插件，并通过 Application Calling 接入现网；
- 插件 / 工具集成：
  - 将 CRM、订单、物流、工单系统封装为 REST API 或 RPC 层，通过百炼「插件」或「应用组件 API」暴露给 Agent 调用；
  - 对写操作类接口（退款、关闭订单等）只提供「生成操作建议 + 工单草稿」，真正执行仍由人工或业务系统审核流驱动；
- 评估与监控：
  - 利用 Model Studio 的统计与监控能力观测调用量、延迟、错误率，结合业务侧埋点计算自助率、AHT、升级率等；
  - 建立小样本的人工评估集，对不同 Prompt/知识库版本/模型进行对比实验。

## 5. 效果与价值（示意）

- 自助解决率提升 20–30 个百分点（依业务差异通常从 30–40% 提升到 50–70%）；
- AHT（Average Handle Time）降低 15–25%，高峰期人工排队时长显著缩短；
- 质检覆盖率从 < 3% 提升到 60–100% 自动质检覆盖，人力质检投入下降但问题发现更及时；
- 新客服平均上手周期缩短 30–50%，话术与合规表现更稳定；
- 对客服对话的结构化数据沉淀，为后续产品优化、运营洞察、风控策略提供输入。

> 注：上述指标为参考范围，具体效果取决于行业类型、流程复杂度、与现有系统集成深度等，可在 POC 阶段通过对照实验和人为评估进行校准。
